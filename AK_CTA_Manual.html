⚡ 速查裁決
CTA 的定位、五級證據、v7 衝突修正與最高 ROI 裁決
核心裁決：CTA = 轉化承接設計（Conversion Handoff），不是演算法推播訊號。CTA 不負責創造流量；CTA 負責承接意圖。

v7 CTA 衝突修正

▼
❌ 原版衝突
Low Friction 範例：留言 +1
把機械式互動誤當低阻力
✅ 升級定義
Low Friction ≠ Low Quality Engagement
低阻力 = 心理阻力低 + 行動成本低 + 非機械互動
🛡 安全 CTA 四大類
不是所有 CTA 都要導向留言；依內容任務選擇最安全且最有效的承接方式

A 類｜真實問題型

用途：留言品質、FB Discussion-Starter、受眾診斷。

你現在卡在哪一步？\n你比較常卡在選題、腳本，還是發布？

B 類｜收藏型

用途：教學內容、檢查表、步驟型內容。

這份清單適合收藏，等你要拍片前再看。

C 類｜私訊承接型

用途：高意圖轉化、DM 診斷、LFA 承接。

如果你也卡在這裡，私訊我你的狀況，我可以幫你看一下。

D 類｜延伸觀看型

用途：長內容導流、系列內容、留存延伸。

下一支我會補第二步，想接著看的人可以先追蹤。

Tier 3 Weak CTA你覺得呢？留言分享你的想法。有需要可以找我。不一定危險，但常因空泛而低轉化。

🔗 CTA → DM → Offer 鏈路
CTA 的任務不是成交，而是讓對的人走到下一個可診斷節點

五節點鏈路

▼
影片 CTA → DM 開場 → DM 診斷 → 方案提出 → 自然成交
節點任務常見斷點
CTA低阻力引導下一步踩 Engagement Bait / CTA 與 Hook 斷裂
DM 開場降低戒心，建立人感直接丟連結、太像機器人
DM 診斷確認對方真正問題沒聽懂需求、過早推銷
方案提出對應問題給解法Offer 不吸引、價值不明
自然成交消除最後阻礙No Trust / No Money / No Hurry 未處理
DM 第一則建議：沒問題，我可以給你。不過我想先確認一下，你現在主要是卡在選題、腳本，還是發布？

📊 Post-check / A/B Test
從「安全」進到「有效」，發布後用數據診斷 CTA 失效原因

CTA 失效診斷

▼
diagnosis意義修正方向
cta_designCTA 本身不清楚、不安全或摩擦過高改寫 CTA 類型或語句
content_value內容本身沒讓觀眾覺得值得行動回頭修 Script / Value
audience_mismatchHook 吸來的人與 Offer 不一致重寫 Hook 與受眾定位
trust_gap信任不足，承諾要求太高降低 CTA 層級，增加 Stories / 過程展示
offer_gap引流品不吸引重新設計 LFA / Micro Offer
timingCTA 出現太早或太晚調整位置與節奏

A/B Test SOP

▼
禁止：不使用危險 CTA 作為新實驗組，例如留言 +1、留言 111、按讚解鎖、打「我」。舊內容可作 archive comparison，但不建議新上線測試。
組別類型CTA
A真實問題型你現在卡在哪一步？留言告訴我。
B私訊承接型想要完整清單，可以私訊我「清單」。
C收藏型這份清單適合收藏，等你拍片前再看。

控制變量：同主題、同 Hook、同 Script、同平台、相近發布時段、同類型受眾，只改 CTA。沒有足夠樣本前，不得把單次波動寫成結論。

🎓 教學驗收
讓團員真正理解：高互動不等於高轉化

學員必懂一句話

▼
CTA 不能無中生有，只能順水推舟。

CTA 是結帳櫃檯，不是百貨公司大聲公。它不能把不想買的人變成買家，只能讓已經心動的人更容易走下一步。
